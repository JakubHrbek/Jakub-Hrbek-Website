CPU Path Tracer

High-performance CPU-based path tracer written in modern C++. Global illumination. Physically-based materials. Multi-threaded rendering pipeline.

CPU Path Tracing Engine

This is a comprehensive CPU-based path tracer written in modern C++17 that simulates global illumination by tracing rays recursively through scenes composed of geometry, materials, and light sources. Built from the ground up without GPU acceleration, it showcases pure computational power and algorithmic efficiency.

The engine uses std::thread for parallelism with tiled rendering, BVH acceleration structures for fast ray traversal, and outputs high-quality PNG images via stb_image_write. Every pixel is calculated through Monte Carlo sampling to achieve photorealistic results.

Advanced Material System

The renderer features a sophisticated material pipeline supporting physically-based rendering (PBR) with metallic/roughness workflows. Materials include dielectric refraction for glass, procedural textures like marble and plasma, and specialized materials including iridescent, chrome, titanium, ice, and concrete surfaces.

Each material implements custom scatter() and emitted() functions with BRDF approximations. The system handles complex lighting interactions including colored lighting, soft shadows from area lights, and realistic surface properties with proper energy conservation.

Performance & Architecture

The core rendering loop builds scenes with camera setup, geometry definition, and BVH construction for spatial acceleration. Multi-threaded tile rendering distributes work across CPU cores, with each pixel receiving stratified sampling and recursive ray bouncing up to maximum depth limits.

Advanced sampling techniques reduce noise while maintaining efficiency. The light manager handles direct lighting queries with shadow rays and BRDF sampling. A procedural skybox provides environmental lighting fallback for rays that miss geometry, creating realistic ambient illumination.

Geometry & Acceleration

The engine supports diverse geometric primitives including spheres, boxes, cylinders, cones, planes, and rectangles, all implementing the hittable interface with bounding box support. A Bounding Volume Hierarchy (BVH) provides logarithmic-time ray intersection testing for complex scenes.

The modular architecture separates concerns with dedicated classes for cameras, rays, materials, lights, and geometry. Custom 3D vector math handles all point, color, and direction calculations. Gamma correction and color clamping ensure proper image output with support for both PNG and PPM formats.
